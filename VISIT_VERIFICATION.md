# FoodMile 방문 인증 — 운영 전 보안 요구사항

현재 구현은 브라우저 GPS, 사진 로컬 미리보기, LocalStorage를 사용하는 프론트엔드 데모다. LocalStorage와 클라이언트 위치 값은 사용자가 변경할 수 있으므로 실제 운영 인증이나 결제성 포인트의 근거로 신뢰하면 안 된다.

운영 전 서버에서 다음을 추가해야 한다.

- 서버가 발급하고 검증하는 timestamp와 일회성 인증 요청 토큰
- GPS 좌표와 accuracy의 서버 전송 및 허용 반경 재계산
- 동일 사용자·동일 매장의 24시간 중복 인증 서버 판정
- 이미지 원본 업로드, 파일 무결성 검사와 메타데이터 검증
- 위치 조작 탐지, 속도·시간·기기 신호를 포함한 부정 이용 방지 정책
- 데모 포인트가 아닌 서버 원장 기반 포인트 지급과 취소 기록
- 위치 좌표, accuracy, 이미지의 보관 기간·삭제·접근 통제 정책

클라이언트는 지속적인 위치 추적이나 이동 경로를 저장하지 않는다. 데모 LocalStorage에는 인증 판정에 필요한 매장 ID, 인증 시각, 단일 위치 좌표, accuracy, 계산 거리와 테스트 여부만 저장하며 주소와 사진 원본은 저장하지 않는다.
